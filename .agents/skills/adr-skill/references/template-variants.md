# Template

Scrollect uses a single lean ADR template: `assets/templates/adr-simple.md`.

This template prioritizes brevity and scannability over exhaustive documentation. All ADRs use the same structure regardless of complexity.

If a decision requires deep option analysis (3+ options with structured pros/cons), use a comparison table inside the "Alternatives considered" section rather than switching to a different template. Keep it concise — the goal is capturing why alternatives were rejected, not exhaustive evaluation.

The MADR-style template (`assets/templates/adr-madr.md`) is retained for reference but not used in this project. If you need it for a different project with heavier governance requirements, it's there.
